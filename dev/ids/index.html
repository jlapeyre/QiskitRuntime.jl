Id Numbers and Tokens

QiskitRuntime.Ids — Module
module Ids

This module implements identification numbers and related objects.

structs implemented here include: JobId, UserId, and Ids.Token.

Note

We may want to expose less from this module than we do at present. For example, generating bogus ids is useful for mocking, but probabaly not for most users.

QiskitRuntime.Ids.JobId — Type
struct JobId

Wrapper type for job ids

Because a session id is the first job id in a batch, JobId is used for session id as well. The data is stored as a string that is validated on construction.

Note that passing validation is a necessary, but not sufficient condition for distinguishing job ids returned by the server. We could investigate the format of job ids and tighten up the validation.

See validate

QiskitRuntime.Ids.Token — Type
struct Token

Wraps authentication tokens.

The server and the web dashboard express tokens as strings of 128 lower-case hexadecimal digits. We store this as eight UInt64s. In particular, the Token is validated upon construction.

Uppercase hexadecimal characters are allowed when parsing.

QiskitRuntime.Ids.UserId — Type
struct UserId

Wraps a user id.

The server returns 24 hex digits. We encode this as 96 bits in three UInt32s. Thus, validation occurs on construction.

QiskitRuntime.Ids.validate — Method
validate(::Type{JobId}, job_id::AbstractString)

Return true if job_id is a valid job id.

job_id must be a twenty digit string of 0-9 and a-z. There may be other restrictions, but we don't know about them and cannot check them.
